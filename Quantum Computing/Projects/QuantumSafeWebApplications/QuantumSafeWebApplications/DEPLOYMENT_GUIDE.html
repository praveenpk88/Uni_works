🐧 Windows Subsystem for Linux (WSL) Setup

WSL allows Windows users to run a Linux environment directly on Windows, providing better compatibility with Linux-based development tools and Docker containers.

🚀 Quick WSL 2 Installation (Windows 10 version 2004+ / Windows 11)

# PowerShell (Run as Administrator) wsl --install # This command will: # - Enable required Windows features # - Download and install Ubuntu (default distribution) # - Set WSL 2 as the default version # - Require a restart # After restart, set up Ubuntu user account # Ubuntu will open automatically and prompt for username/password

📋 Manual WSL 2 Installation (Older Windows 10 versions)

# Step 1: Enable WSL feature (PowerShell as Administrator) dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart # Step 2: Enable Virtual Machine Platform dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart # Step 3: Restart your computer shutdown /r /t 0 # Step 4: Download and install WSL2 Linux kernel update # Visit: https://aka.ms/wsl2kernel # Download and run the MSI installer # Step 5: Set WSL 2 as default version wsl --set-default-version 2

🐧 Install Linux Distribution

# List available distributions wsl --list --online # Install Ubuntu (recommended for development) wsl --install -d Ubuntu # Alternative distributions wsl --install -d Ubuntu-20.04 wsl --install -d Ubuntu-22.04 wsl --install -d Debian wsl --install -d kali-linux

⚙️ Configure WSL 2

# Check WSL version and status wsl --list --verbose # Set specific distribution to WSL 2 (if needed) wsl --set-version Ubuntu 2 # Launch your Linux distribution wsl # Or launch specific distribution wsl -d Ubuntu

🔧Troubleshooting

🔍 Common Issues & Solutions

❌ Issue: Port already in use
🔧 Solution:

🍎 macOS / Linux

# Find and kill process using port 5000 lsof -ti:5000 | xargs kill -9 # Alternative: Find process first, then kill lsof -i:5000 sudo kill -9

🏢 Windows

# PowerShell - Find and kill process using port 5000 $processId = (Get-NetTCPConnection -LocalPort 5000).OwningProcess Stop-Process -Id $processId -Force # Alternative: Using netstat and taskkill netstat -ano | findstr :5000 taskkill /PID /F # Command Prompt version for /f "tokens=5" %a in ('netstat -aon ^| find ":5000" ^| find "LISTENING"') do taskkill /f /pid %a
❌ Issue: Database connection failed
🔧 Solution:
# Check database status docker-compose exec postgres pg_isready -U quantum_user # Restart database service docker-compose restart postgres
❌ Issue: Out of disk space
🔧 Solution:
docker system prune -a --volumes
